# UX & A11y Notes

> **대상**: 디자이너, 프론트엔드, 심사위원  
> **목적**: 시니어 친화적 UX 원칙 및 접근성 가이드

---

## 👥 연령대별 UX 포인트

### 50대 (1970-1979년생)

**특징**:
- 디지털 기기 사용 경험 있음
- 스마트폰 앱 설치/삭제 가능
- 긴 텍스트 읽기 가능

**어려움**:
- 작은 폰트 (16pt 이하)
- 복잡한 네비게이션 (3단계 이상)
- 영어 전문 용어

**대응**:
- 기본 모드 (normal) 제공
- 명확한 아이콘 + 텍스트 레이블
- 한글 우선, 영어는 괄호 안에

---

### 60대 (1960-1969년생)

**특징**:
- 스마트폰 기본 기능 사용
- 카카오톡, 유튜브 익숙
- 자녀/손주와 소통 의지 높음

**어려움**:
- 멀티태스킹
- 복잡한 설정
- 에러 메시지 이해

**대응**:
- 한 화면 한 가지 행동
- 에러 메시지에 해결 방법 명시
- "쉬움" 모드 (easy) 제공

---

### 70대 (1950-1959년생)

**특징**:
- 스마트폰 최근 시작
- 전화, 문자 위주 사용
- 가족 도움 필요

**어려움**:
- 작은 버튼 (44pt 이하 터치 어려움)
- 빠른 애니메이션
- 시력 저하 (대비 낮은 색상)

**대응**:
- "초대형" 모드 (ultra) 제공
- 애니메이션 최소화
- 고대비 색상 (4.5:1 이상)

## 🎨 UI 원칙

### 1. 한 화면 한 가지 주요 행동

❌ **나쁜 예**:
```
[홈 화면]
- 오늘의 카드
- 인사이트 3개
- Q&A 5개
- 도구 트랙
- 포인트/배지
```

✅ **좋은 예**:
```
[홈 화면]
- 오늘의 카드 (큰 카드)
- 하단 탭바: 홈 / 인사이트 / 커뮤니티 / 더보기
```

---

### 2. 짧은 문장, 어려운 용어 피하기

❌ **나쁜 예**:
- "생성형 AI 모델의 하이퍼파라미터 튜닝"
- "클라우드 스토리지 동기화 실패"

✅ **좋은 예**:
- "AI가 글을 쓰는 방법"
- "사진 저장이 잘 안 됐어요. 다시 시도해주세요."

---

### 3. 버튼/폰트 크기/대비 기준

| 모드 | 폰트 (body) | 버튼 높이 | 아이콘 | 간격 |
|------|------------|----------|--------|------|
| normal | 16dp | 48dp | 24dp | 16dp |
| easy | 20dp | 56dp | 28dp | 20dp |
| ultra | 24dp | 64dp | 32dp | 24dp |

**터치 영역**: 최소 44 × 44dp (Apple HIG), 48 × 48dp (Material Design)

**색상 대비**: WCAG 2.1 AA 기준 (4.5:1 이상)
- `#000000` (검정) on `#FFFFFF` (흰색) = 21:1 ✅
- `#666666` (회색) on `#FFFFFF` = 5.74:1 ✅
- `#999999` (밝은 회색) on `#FFFFFF` = 2.85:1 ❌ (caption만 사용)

## 📝 텍스트 작성 가이드

### Daily Card

- **길이**: 300-500자
- **문장**: 20자 이하로 끊기
- **톤**: 친절하지만 과장 없이
- **예시**:
  ```
  AI(인공지능)는 컴퓨터가 사람처럼 생각하고 배우는 기술이에요.
  
  예를 들어, 음성 비서(시리, 구글 어시스턴트)가 여러분의 말을 
  알아듣고 대답하는 것도 AI 덕분이에요.
  
  요즘은 AI가 사진을 분석해서 꽃 이름을 알려주거나, 
  문자 메시지를 읽어주기도 합니다.
  ```

### Quiz

- **문항 수**: 1-3개
- **보기**: 4지선다
- **정답 설명**: 1-2문장
- **예시**:
  ```
  Q: AI가 할 수 있는 일은 무엇인가요?
  
  A) 사진 속 꽃 이름 알려주기
  B) 날씨 예보하기
  C) 문자 메시지 읽어주기
  D) 모두 가능해요 ✅
  
  설명: AI는 사진 분석, 음성 인식, 추천 등 다양한 일을 할 수 있어요!
  ```

### Insight

- **길이**: 500-800자
- **제목**: 10자 이내
- **요약**: 50자 이내
- **톤**: 정보 전달 위주, 중립적

### Error Messages

❌ **나쁜 예**:
- "Network timeout (ERR_CONNECTION_TIMED_OUT)"
- "Invalid token"

✅ **좋은 예**:
- "인터넷 연결이 불안정해요. 와이파이나 데이터를 확인해주세요."
- "로그인 시간이 만료됐어요. 다시 로그인해주세요."

## 🔊 TTS 사용 원칙

### TTS 제공 화면

- ✅ Daily Card 본문
- ✅ Insight 본문
- ✅ Q&A 질문/답변
- ❌ 버튼 레이블 (VoiceOver/TalkBack으로 충분)
- ❌ 에러 메시지 (화면에 표시만)

### TTS 버튼 위치

```
[카드 제목]
🔊 [읽어주기]
────────────
[카드 본문]
────────────
```

### TTS 속도

- 기본: 0.9x (약간 느리게)
- 설정 가능: 0.7x / 0.9x / 1.0x

## ♿ 접근성 체크리스트

### Web (WCAG 2.1 AA)

- [ ] 모든 이미지에 `alt` 텍스트
- [ ] 키보드만으로 전체 네비게이션 가능
- [ ] 폼 입력란에 `<label>` 연결
- [ ] 색상 대비 4.5:1 이상
- [ ] 폰트 크기 브라우저 확대 시 깨지지 않음

### Mobile (iOS/Android)

- [ ] VoiceOver/TalkBack 지원
- [ ] `accessibilityLabel` 모든 버튼에 명시
- [ ] 터치 영역 최소 44dp
- [ ] 애니메이션 과도하지 않게 (motion sickness)
- [ ] 고대비 모드 (ultra) 제공

## 🎨 Color Palette

### Primary Colors

```
Primary Blue: #2196F3
  - 주요 버튼, 링크

Success Green: #4CAF50
  - 성공 메시지, 복약 체크 완료

Warning Orange: #FF9800
  - 경고 메시지, 사기검사 "경고"

Danger Red: #F44336
  - 에러 메시지, 사기검사 "위험"
```

### Text Colors

```
Primary Text: #000000 (검정)
  - 제목, 본문

Secondary Text: #666666 (회색)
  - 설명, 메타 정보

Tertiary Text: #999999 (밝은 회색)
  - Caption, 비활성

Link: #2196F3 (파랑)
  - 링크, 버튼
```

### Background Colors

```
Background: #FFFFFF (흰색)
Card: #F5F5F5 (연한 회색)
Info: #E3F2FD (연한 파랑)
Warning: #FFF3E0 (연한 주황)
Error: #FFEBEE (연한 빨강)
Success: #E8F5E9 (연한 초록)
```
